# Project Instructions

- Refine the UI instead of replacing the product direction.
- Keep the current project color palette as the default visual language unless the user asks otherwise.
- Do not use mocked, fake, placeholder, or hardcoded demo data in product flows.
- Prefer real Supabase-backed data. If data is unavailable, use loading, empty, and error states instead of sample records.
- When the user asks for layout fidelity, prioritize matching the provided design structure while preserving this project's palette and product context.
